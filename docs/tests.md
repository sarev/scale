# Tests: the model-free regression suite

`tests/` holds fast, **model-free** regression tests â€” the LLM is stubbed and only parsing/patching/caching run, so
the suite finishes in seconds. Run all with `env/Scripts/python.exe tests/run_all.py`, or any single
`tests/test_*.py` directly. Each guards a specific past bug; keep them green when touching the relevant area, and
add one when fixing a new bug.

`tests/dogfood.py` is the **model-dependent** end-to-end harness: it walls every `scale*.py` module (docstrings,
comments, and intra-routine blank lines stripped) into `temp/dogfood/walled/`, runs the current SCALE over copies in
`temp/dogfood/annotated/` (`--mode offline` for the pure local pipeline, `--mode escalation` to emit the run manifest
for the /scale loop), then deterministically verifies that re-walling each output reproduces the input's code lines
exactly. Both directories are kept for review; `--files`/`--wall-only` scope it down. Slow on the full set â€” it loads
a real GGUF.

## Catalogue

- `test_summary_injection.py` â€” the whole-file summary is actually injected into the LLM context. Guards the critical regression where a stale `"OK"` was interpolated instead of the summary, silently disabling the feature.
- `test_file_identity.py` â€” `_file_identity_note` classifies header vs implementation vs other (case-insensitive suffix), and `prime_llm_for_comments` injects the file-name/role turn before the file-overview turn (so the summary and routine/block turns all see it).
- `test_doc_style_and_delist.py` â€” the file-description de-listing (`_looks_listy`/`_reflow_if_listy`/`_strip_list_markers`: a listy summary is reflowed by one model turn, falling back to deterministic marker-stripping; flowing prose is left alone) and the C/JS doc-comment style detection (`_detect_doc_style_c`/`_js`: leading banner ignored, `//`-bodied â†’ line, mixed/block â†’ block) + line-comment rendering and the `style="line"` patcher path.
- `test_cache_invalidation.py` â€” `SummaryCache` reuses on identical content but regenerates when the source changes (content-hash keying), so edits don't get a stale summary.
- `test_qualname_collision.py` â€” same-named definitions get distinct docstrings. Guards against doc maps keyed by qualname (now keyed by node identity).
- `test_inline_def.py` â€” inline one-line `def`/`class` are skipped rather than corrupted, and output stays valid Python.
- `test_cr_line_endings.py` â€” bare `\r` (old-Mac) line endings still map tree-sitter rows to the right source lines, in both the C and JS workers.
- `test_js_imports.py` â€” CommonJS `require()` (including destructured forms) is parsed via the grammar's `arguments` field rather than fragile index traversal.
- `test_js_extractor.py` â€” the JS comment extractor accepts a `/** */` block, a ``` fence, or plain text, and preserves genuine content bullets instead of eating leading asterisks.
- `test_elision.py` â€” `elide_to_budget` preserves the header, fits the token budget, and reports the omitted-line count (degrades to header-only on a tiny budget).
- `test_chunk_split.py` â€” `_split_source` covers every line exactly once (chunks rejoin to the original), keeps each chunk within budget, and hard-splits over-long single lines.
- `test_mapreduce_summary.py` â€” `_generate_file_summary` uses one pass when the file fits and switches to map-reduce (multiple map calls + a reduce) when it doesn't, ending the large-file path with a final description-shaping turn.
- `test_block_boundaries.py` â€” the Python block provider numbers only legal statement starts at all depths, excludes continuation/`a; b`/inline-compound lines and leading docstrings, and does not descend into nested defs (which become single opaque boundaries); classes get a block pass too. `test_block_boundaries_c.py` / `test_block_boundaries_js.py` assert the same for the tree-sitter providers (JS additionally: a nested function is one opaque boundary and its own target).
- `test_structural_breaks.py` â€” the shared rule engine (`structural_breaks`) tested on hand-built `SegStatement` records: each rule (first-in-scope/after-def/before-return/before-compound/dedent) fires or abstains correctly, a `merge_anchor` redirects a trailing return's break to its anchor, the opening paragraph is always emitted as a chunk (when the first statement is a legal boundary), the `allow_*` flags gate the brace-language differences, and breaks land only on legal boundaries with well-formed ranges. `test_structural_segments_c.py` / `test_structural_segments_js.py` check each worker's adapter end-to-end on a crafted function (C: before-compound/return/dedent, no after-def; JS: after-def fires).
- `test_block_merge_return.py` â€” a `[simple_stmt, return]` suite merges into one paragraph anchored at the leading statement (the return gets no own break, so the comment pass sees both lines), the patcher inserts no blank between the block opener and the merged comment, a many-statement body's trailing return still splits, and code is preserved â€” across Python, C, and JS.
- `test_block_decl_separation.py` â€” the leading-declaration heuristic: a scope opening with â‰Ą2 variable declarations (C `declaration`, JS `let`/`const`/`var`) breaks before its first real statement so the declarations form their own uncommented block; a single leading declaration does not trigger it; code preserved (C + JS).
- `test_comment_scoring.py` â€” the two-turn comment pass (via a fake LLM): the parsers (`_parse_summary`/`_parse_score` â€” 1â€“3, clamped so a stray 4/5 reads as 3/`_comment_to_insert`) behave, turn 1 always yields a summary (nudged once if unusable), a high score returns a clean insertable line while a low (restatement) score suffixes `VALUE_FLAG` (kept for context, skipped at output), the `value_threshold` (`--block-comments`, default `medium`=2) moves the cut, and the persistent message list is restored.
- `test_block_insertion.py` â€” the insertion patcher inserts blank+comment above chosen chunks, replaces an existing same-indent comment, treats `NONE` as a true no-op (code/blanks/existing comment untouched), preserves the code signature, aborts the routine (original kept) when a forced edit would alter code, and round-trips a `//` line-comment language (C/JS) too.
- `test_block_numbered_view.py` â€” over-long non-boundary runs collapse to an elision band carrying no number (short runs shown verbatim); the segment parser keeps only ranges whose start is a legal boundary, clamps ends within the body / before the next chunk, and falls back to bare numbers as starts.
- `test_escalation_routing.py` â€” `cognitive_complexity` ranks routines sensibly (nesting penalty, elif as a cheap continuation, opaque nested defs), the `Escalation` cutoff is strict (`>`) with a codestats-JSON override winning per-qualname, and `node_sig` is stable across the line shifts between emit and apply but changes when code changes.
- `test_cognitive_c_js.py` â€” the native C/JS cognitive scorers (`cognitive_complexity_c`/`cognitive_complexity_js`) mirror the Python rules (nesting penalty, `else if` folded as a cheap continuation, run-collapsed boolean sequences, opaque nested JS defs, try/catch/finally parity, expression-bodied arrows scored, bodyless C prototypes 0), the C/JS block providers stamp the score onto `BlockTarget.cognitive`, and the C def pass defers a routine on its native score alone (no codestats report).
- `test_def_snippet_comments.py` â€” the Python def-pass snippet (assembled by the real `generate_docstrings` with a fake model) preserves the author's standalone in-body comments, including ones sitting between statements (the gap the old statement-by-statement assembly dropped).
- `test_docstring_recovery.py` â€” priming never makes the model parrot "OK" (a fixed `PRIMING_ACK`, no "say OK" asks, summary is the only priming generation); the def pass nudges an unusable reply once, then promotes the routine to the manifest when escalation is active (else uses the placeholder), and uses a good reply as-is with no nudge.
- `test_structural_elision.py` â€” `summarise` caps the reply per requested length and weaves subject+text into the prompt; `elide_structurally` leaves a fitting snippet untouched, on a tight budget collapses the deepest body suite first into a `...  # summary` (keeping its header and the shallower statements) staying valid Python and within budget, and falls back to the crude crop on unparseable input.
- `test_manifest_roundtrip.py` â€” emit defers a complex routine (recording its verbatim span once, with the block chunks as `bidx` + line ranges INTO that snippet) and leaves it byte-for-byte untouched while annotating a simple one locally; both passes merge into ONE per-routine request; the model-free apply re-binds and patches a stronger model's answers (block by boundary index, doc by `(qualname, sig_hash)`), honours `NONE`/unfilled answers, and preserves all original code.
- `test_manifest_c.py` â€” C escalation: a doc-site redirected prototype is always deferred when a manifest is active (its snippet is the impl body), a codestats-scored routine's block recipe defers with the routine untouched, `run_manifest` stamps `file` and collapses byte-identical snippets into `snippet_ref` (the code crosses the wire once), `unfilled_answers` counts the open slots, and the model-free `apply_manifest_c` patches the prototype doc above the declaration and the block comments by boundary index, code preserved.
- `test_filedoc_license_guard.py` â€” the file-doc license veto (`looks_legal`) flags copyright/SPDX/license-name/warranty text and leaves plain description prose editable (the safety net behind the model's classification).
- `test_filedoc_classify_parse.py` â€” `_parse_classify_range` parses `START-END`/single/`NONE`, clamps to the eligible-entry count, and rejects an inverted or empty range (so a confused model can't point the patcher outside the header).
- `test_filedoc_zone_c.py` â€” `scale_c.file_doc_target_c` gathers a **multi-block** leading zone (a `/* */` block, a blank, then `//` lines all count), reconstructs the per-line decoration prefix, stops the scan at the first `#include`, and reports no zone (fresh-insert) for a code-first file.
- `test_filedoc_py.py` â€” Python file-doc targets the **module docstring**: an existing description is replaced (licence inside the docstring preserved, existing text seeds the provider), a licence-only docstring gets a description appended (veto â†’ no seed), a file with no docstring gets a fresh one inserted as the first statement, code is unchanged (`_module_code_signature`), and `_py_doc_preserved` rejects a code change or a docstring-breaking description.
- `test_filedoc_js.py` â€” JS file-doc (shared brace scanner) replaces a header-block description keeping the licence, and inserts a fresh `/* */` block for a code-first file; code preserved.
- `test_filedoc_patch_c.py` â€” end-to-end via a fake LLM (the description prose comes from a stub `summary_provider`, no generate turn): an existing description is replaced in place while the copyright/license block stays byte-for-byte and the existing text is handed to the provider as a seed; a license-only header has a description appended (the model's misclassification of the license lines as the description is vetoed â†’ no seed); a code-first file gets a fresh `/* */` block; and `file_doc_preserved` turns a code-altering splice into a no-op.
- `test_filedoc_summary_seed.py` â€” the whole-file summary is generated to the file-description spec, and a supplied existing-description seed is woven into the generation prompt (and omitted when there's no seed).
- `test_run_model.py` â€” the multi-file run model: `gather_files` expands dirs (by source extension) / globs / explicit files with dedup+ordering; `scan_run_files` loads/parses each run file exactly once into the retained store (target/reference flags, language, symbols with spans; a duplicated path scanned once); targets parse as a list and `--reference` accumulates; `main` fails fast (before any model load) on no matches and `-o` with multiple targets (annotate and apply alike); `--check-manifest` needs no targets and exits 1 with unfilled answers / 0 when complete; and the call-graph pre-pass (`_build_call_graph` over the store) resolves a cross-file call, seeds the store, and orders the callee's file first.
- `test_call_graph.py` â€” the call-graph layer (`scale_project`, pure/model-free): the confident-only resolver (free same-file-first then run-wide-unique, ambiguous unresolved; `self`/`this`â†’enclosing class; unique-method-name link; ambiguous method-name unresolved) over both 2-tuple and widened `(name, kind, line)` call records, the per-call-site `call_map` (unresolved calls absent), leaf-first ordering with nesting+call edges, recursion / mutual recursion collapsing into a terminating SCC, the coarse `file_order` (callee's file first, unconstrained files keep input order), `ContractStore` (seed from existing docs, `update` first-line, `callee_notes` caps and omits contract-less callees, the `contract` getter, `missing_callee_contracts` as the lazy generator's work list), and `apply_doc_order`.
- `test_iter_symbols.py` â€” each worker's `iter_symbols` (Py/C/JS) extracts the parent qualname, the full `start`/`end` span, the existing-doc seed, and classified call sites as `(name, kind, line)` triples (`free`/`self`/`method`, with the call's 1-based line), walking only the routine's own body so a nested def's calls belong to it, not the enclosing routine (a class body has no own calls; a JS `var_arrow`'s body is walked since its value is not a separate def node).
- `test_callgraph_wiring.py` â€” the def-pass call-graph hooks via a fake LLM (Py/C/JS): routines are visited in `doc_order`, each generation turn carries its callees' contract notes (`callee_context`), `on_doc` fires after each routine, and absent the hooks the loop is unchanged (deepest-first, no injected notes).
- `test_lazy_callee_oneliner.py` â€” the lazy callee one-liner generator (`_make_callee_oneliner_context`, via a fake LLM): documenting a caller generates a missing callee's one-liner from its body in the retained store (one model call), caches it on the `ContractStore` (no second call for later callers), never summarises an uncalled or already-documented (seeded) routine, and does not retry a callee that yielded nothing.
- `test_block_callee_annotation.py` â€” the block pass's read-side call annotations: `_block_callee_notes` re-parses the CURRENT (def-pass-shifted) text and places "callee: one-liner" notes on the calls' current lines via `call_map`; `request_block_comment(line_notes=...)` appends the note to the call line (and only the call line) of the paragraph the model reads; and `annotate_blocks(callee_annotations=...)` never writes the note to the output (code untouched).
- `test_def_doc_seed.py` â€” the def pass's ingest-and-update seed: a C/JS routine with a doc comment above its header (outside the snippet) has that doc surfaced in its generation turn as an "already documented â€” ingest and update" block (and an undocumented routine gets none); Python's assembled snippet carries its existing docstring natively.
- `test_project_blurb.py` â€” project context: `find_project_doc` prefers CLAUDE.md / matches README variants / walks up to the `.git` root; `resolve_project_doc` honours `none`/explicit/auto; `project_blurb` distils-and-caches (no regen on the second call); the blurb is injected before the file-summary turn.
- `test_comment_trailing_space.py` â€” generated doc-comments carry no trailing whitespace across every renderer (C/JS `_render_*_block`, Python `patch_docstrings_textually`); a blank continuation line is `" *"`, never `" * "`.
- `test_doc_site_c.py` â€” the C header/implementation doc-site (`--doc-site`): `iter_decls_with_info_c` collects prototypes (multi-line, `#ifdef`-nested, pointer-return) and ignores variable/typedef/function-pointer/local declarations; `plan_doc_sites_c` pairs a unique def with its target header (skip + ordering pair + prose-from-body), never skips under `impl`, leaves a decl-only/ambiguous name un-skipped, and never writes a reference; the C worker (via a fake LLM) documents the prototype from the impl body and leaves the redirected definition byte-for-byte untouched, `iter_block_targets_c`'s `doc_override` restores the header doc into the block context, and `_order_header_before_impl` moves a header ahead of its impl.
- `test_verify_gate.py` â€” the def-pass verification floor (via a fake LLM with clean-context challenge routing): `ungrounded_tokens` flags invented backticked identifiers (and only those â€” grounded names, single chars, and unbackticked prose pass), the gate's one nudge accepts a corrected regeneration, the grounding challenge parses NONE vs a verdict and regenerates once with the verdict as feedback, a twice-failed docstring is promoted to the manifest (the routine byte-for-byte untouched, its snippet recorded) when escalation is active and written under a warning when not, and the message list stays balanced.
- `test_verify_blocks.py` â€” the block-pass verification floor: an obviousness NO regenerates the note once (re-scored, re-challenged) and a second NO tags it `CHALLENGE_FLAG` (kept as context, never written); the per-note grounding gate nudges then flags; the story challenge fires only above `SHORT_FUNCTION_CHUNKS` chunks with at least one insertable note, regenerates the whole note set once on RESTATE, and on a second RESTATE drops the routine's comments (paragraphing blanks stay) or â€” with a manifest â€” promotes the routine whole (chunk recipe recorded, routine untouched); a single twice-failed note also promotes.
- `test_skeleton.py` â€” the model-free skeleton + two-pass descriptions: `render_skeleton` keeps the leading zone, signatures (methods indented under their class), existing docs, and C `#define`s (multi-line macro bodies elided; same-file prototypes deduplicated) while bodies never appear; the binary guard returns None for a no-symbol file; pass-1 priming summarises the skeleton (never bodies); and the file-doc pass runs LAST, generating the published description from the ANNOTATED text's skeleton (it sees the def pass's fresh docstrings).
- `test_reword.py` â€” the header-reword manifest: `apply_reword` locates the draft by exact (whitespace-collapsed) match and replaces it re-decorated in the block's own prefix with licence lines untouched; an unmatched draft and a `"NONE"` answer are safe no-ops; the legal veto refuses a legal-smelling match; Python rewords through its parse-based guard (a docstring-breaking answer is rejected); `unfilled_rewords` counts empty answers; and the manifest I/O round-trips, rejecting non-reword files.

## Model-dependent eval harnesses (`tests/block_eval/`)

Not run by `run_all.py` â€” it's a subdirectory, and the `test_*.py` glob is non-recursive. For eyeballing block-pass
quality as the model/prompts change: `make_wall.py` (strip a file to a docstring-only wall), `show_segments.py` /
`show_comments.py` (print the segmenter's chunks / the per-chunk comments without patching), and `samples/`
fixtures. They load a real GGUF (`SCALE_MODEL` overrides the default). Also here: `segment_eval.py`, the
**model-free** harness used to develop the structural segmenter â€” it walls a corpus, re-segments it
deterministically, and diffs/scores against the human paragraphing (`--maps` for per-routine break maps). See
`tests/block_eval/README.md`.
